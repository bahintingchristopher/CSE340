You are definitely on the right track, Chris! You've hit on the exact core concept: security and protecting your application from malicious input.

To explain it a bit further, using $1 is a technique called a parameterized query (or prepared statement).

When you use a placeholder like $1 and pass the value in a separate array, the database treats that value strictly as data, never as executable code. If someone tried to pass malicious SQL commands into categoryId, the database wouldn't run it; it would just look for a category ID that literally matches that weird text.

While it protects your overall application from data manipulation rather than HTML directly (that's usually handled in the view layer), it is the gold standard for stopping a highly dangerous attack called SQL Injection.

Spot on, Chris! It all comes back to security.

The <%= %> tag is used for HTML escaping. If category.name somehow contained malicious HTML or a script (like <script>stealData()</script>), <%= %> converts those symbols into harmless text characters (&lt;script&gt;) so the browser doesn't execute it. On the other hand, <%- %> outputs the raw, unescaped string. You intentionally used it for your header and footer partials because those files contain actual HTML tags that the browser needs to render as real layout elements.
